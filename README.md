
 ROOMS ESCAPE (Rust日本語版)

Rustの軽量・高速なGUIライブラリ `egui` (および `eframe`) を使用して開発された、シンプルな2D脱出ゲームエンジンです。
Java Swingで実装されたオブジェクト指向の設計を、Rustの「所有権システム」や「列挙型（Enum）」を活かした安全かつモダンなアーキテクチャに移植しました。

## 🛠️ 主な機能と特徴

- **完全日本語対応**: コンパイル時に日本語フォント（Noto Sans等）をバイナリ内へ埋め込むことで、RustのGUIで発生しがちな「文字化け（□□表示）」を完全に防止しています。
- **イミディエイト・モード（Immediate Mode）**: 毎フレーム自動で高速再描画を行うゲームループを採用。手動での画面更新（`repaint`など）が不要で、直感的かつスムーズな描画・イベント処理を実現しています。
- **インベントリ＆ギミック管理**: アイテムの「選択状態」をインデックスで安全に管理し、特定のアイテムを所持・選択している時だけ作動する「鍵」や「4桁の暗証番号入力パネル（ダイヤル錠）」などの定番ギミックを搭載しています。

## 📦 動作環境・準備

### 1. フォントファイルの配置
コンパイル時に読み込むための日本語フォントファイル（例: `NotoSansJP-Regular.ttf` など）を用意し、**`Cargo.toml` と同じ階層（プロジェクトルート）に配置**してください。

> 💡 **注意**: コード内では `../NotoSansJP-Regular.ttf`（`src/` から見た相対パス）を読み込む設定になっています。別のフォント名を使用する場合は、`src/main.rs` 内の `include_bytes!` マクロのファイル名を書き換えてください。

### 2. 起動コマンド
ターミナルで以下のコマンドを実行すると、自動的にコンパイルが始まりゲームが起動します。

```bash
cargo run
🎮 ゲームの遊び方・攻略フロー
ゲームは3つの部屋（部屋A・部屋B・部屋C）で構成されています。

スタート（部屋A / 青背景）

最初はパスワードもカードキーもないため、脱出扉は開きません。まずは画面上部をクリックして「部屋C」へ移動します。

アイテムの取得（部屋C / オレンジ背景）

白い「机の引き出し」をクリックして 「小さな鍵」 を手に入れます。

壁に書かれた落書きのヒント（ごくろうさん = 5963）を確認し、画面下部をクリックして部屋Aに戻ります。

ギミックの解除（部屋B / ピンク背景）

部屋Aから右クリックで「部屋B」へ移動します。

画面下のアイテム欄から「小さな鍵」をクリックして選択状態（黄色に変化）にします。

その状態で「クローゼット」をクリックすると鍵が消費され、中から 「電子カードキー」 が手に入ります。

脱出（部屋A / 青背景）

部屋Aに戻り、今度は「電子カードキー」を選択状態（黄色）にします。

画面左の赤い「脱出扉」をクリックすると、暗証番号入力パネルが起動します。

各数値をヒント通り 5 9 6 3 に合わせると、脱出成功（エンディング画面）となります！

📂 コードの構成
src/main.rs: 状態管理、UI描画、ゲームロジックがすべて1ファイルに集約されており、可読性・拡張性に優れています。

Cargo.toml: 依存関係（eframe = "0.26"）の管理。
